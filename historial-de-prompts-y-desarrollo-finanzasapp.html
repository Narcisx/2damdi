Historial de Prompts y Desarrollo - FinanzasApp

A continuación se detalla la lista de prompts y acciones clave realizadas durante el desarrollo de la aplicación, siguiendo el estilo solicitado.

1. Inicio del Proyecto y Arquitectura

Prompt: "Crear una aplicación web moderna para gestión de finanzas personales. Debe usar React, Vite, TailwindCSS y Supabase para el backend. Quiero que sea responsive y tenga un diseño profesional."

Inicié el proyecto configurando el entorno con Vite para asegurar un rendimiento rápido. Elegí TailwindCSS para un estilizado ágil y moderno, y Supabase como solución integral para base de datos y autenticación, estableciendo una arquitectura escalable desde el primer momento.

2. Autenticación y Seguridad

Prompt: "Implementar sistema de Login y Registro con correo electrónico usando Supabase Auth. Los usuarios solo deben poder ver sus propios datos."

Desarrollé el módulo de autenticación integrando los hooks de Supabase. Lo más importante fue la configuración de Row Level Security (RLS) en la base de datos, garantizando que, aunque todos los datos estén en la misma tabla, cada usuario tenga un acceso blindado y exclusivo a su información.

3. Gestión de Transacciones y Categorías

Prompt: "Necesito poder añadir ingresos y gastos. Cada transacción debe tener monto, descripción, fecha y categoría. Quiero ver un resumen en el panel principal."

Creé las tablas transactions y categories y las conecté mediante claves foráneas. Implementé un Dashboard interactivo que calcula en tiempo real el balance total, ingresos y gastos, diferenciando los flujos de dinero visualmente con colores (verde para ingresos, rojo para gastos).

4. Módulo de Ahorro Inteligente

Prompt: "Añadir una sección de Ahorro. Quiero que al gastar, pueda redondear la cifra y mandar la diferencia a una hucha virtual."

Diseñé una lógica de "hucha virtual" en el archivo 

 

Savings.tsx. Implementé algoritmos para calcular redondeos automáticos en las compras y visualizar el progreso de ahorro con gráficos atractivos, simulando un producto bancario real.

5. Almacenamiento de Recibos (Storage)

Prompt: "Quiero poder subir una foto del ticket o factura cuando creo una transacción."

Configuré un Storage Bucket en Supabase llamado 'receipts'. Tuve que ajustar las políticas de seguridad (Policies) varias veces mediante scripts SQL para permitir la subida de cualquier tipo de archivo (imágenes, PDFs) y asegurar que los usuarios pudieran visualizar sus propios documentos.

6. Sistema Bizum (Transferencias Instantáneas)

Prompt: "Implementar una función estilo Bizum. Un usuario debe poder enviar dinero a otro usando un código único, y la transferencia debe ser atómica."

Esta fue una de las características más complejas. Creé una función RPC en base de datos (send_bizum) para manejar la transacción de forma "atómica" (o se hace todo o no se hace nada). Generé códigos únicos (ej. BZ-1234) para cada usuario y diseñé una interfaz fluida para el envío de dinero.

7. Generación de APK Android

Prompt: "Quiero convertir esta web en una App móvil para Android. Configura Capacitor y genera el APK."

Integré Capacitor en el proyecto React. Ajusté archivos de configuración como 

 

build.gradle y capacitor.config.json para que la web funcionara nativamente en Android, gestionando permisos y el área segura (Safe Area) de los dispositivos móviles.

8. Internacionalización (i18n)

Prompt: "La aplicación debe estar en español por defecto, pero preparada para traducirse."

Implementé i18next para gestionar los textos. Extraje todas las cadenas de texto a archivos JSON separados, lo que permite cambiar el idioma de la aplicación instantáneamente y facilita el mantenimiento a largo plazo.

 

9. Automatización Total de Fecha y Hora

Prompt: "Quiero que la fecha y hora sean automáticas al momento de la operación. Elimina el selector de fecha para que sea más rápido."

Para maximizar la precisión y usabilidad, eliminé el campo de fecha manual. Migré la base de datos de DATE a TIMESTAMP para soportar horas y minutos, y modifiqué el código para capturar el instante exacto (CURRENT_TIMESTAMP) en que se realiza la operación, evitando manipulaciones o errores manuales.

Funciones Técnicas Destacadas

  1. send_bizum (RPC Database): Función SQL que orquesta el movimiento de dinero entre dos usuarios, verificando saldos y existencia de usuarios en milisegundos.
  2. useAuthStore (Zustand): Gestor de estado global que mantiene la sesión del usuario activa y accesible desde cualquier parte de la app sin recargar.
  3.  
    TransactionForm Dinámico: Un formulario inteligente que adapta sus campos y validaciones automáticamente dependiendo de si es un Gasto o un Ingreso.
  4. Políticas RLS (Row Level Security): El "cortafuegos" de la base de datos que impide por diseño que un usuario pueda siquiera consultar datos que no le pertenecen.

Tecnologías Utilizadas

  • Frontend: React, TypeScript, TailwindCSS, Vite.
  • Backend/DB: Supabase (PostgreSQL), Supabase Auth, Storage.
  • Móvil: Capacitor (Android).
  • Estado: Zustand.
  • Internacionalización: i18next.
  • Iconos: Lucide React.
