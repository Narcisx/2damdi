Visor de archivos local

Visor de archivos local

  • Prompts
  • Scripts

En este post encontraras los prompts utilizados para crear esta app:

Prompts

    1. Eres un desarrollador experto en HTML, CSS y JavaScript puro (Vanilla JS). Crea una aplicación web limpia y modular (en un solo archivo index.html o separados) que permita subir y visualizar archivos locales.
    2. "Ahora quiero mejorar el estilo de la aplicacion con un diseño moderno y oscuro, que  pueda leer archivos .zip, En lugar de una lista plana, quiero un 'Tree View' (Vista de Árbol) interactivo.
    3. Ahora quiero que soporte archivos multimedia, también debe soportar código (Python, java etc) y poder ejecutarlo en medida de lo posible, imágenes, PDFs.
    4. Reproductor Multimedia "Cyberpunk":"Ahora hagamos un reproductor chulo y personalizado con efectos para audio y vídeo. Quiero que tenga un visualizador de audio con barras que se muevan y controles flotantes."
    5. Mejora Visual del Botón de Tema (SVG):"Haz que el botón de modo claro/oscuro sea un logo SVG animado (un sol y una luna) que rote y cambie al pulsarlo, en lugar de texto simple."
    6. Ajuste de Tamaño (Full Screen)"Haz que la visualización de los archivos (HTML, PDFs, vídeos) ocupe el 100% del tamaño disponible en la pantalla, en lugar de verse pequeños o centrados en una caja."
    7. Efecto "Aura" en la Barra de Estado:"Quiero que el indicador de 'Listo' tenga un aura o resplandor animado que cambie de color dinámicamente dependiendo del tipo de archivo que carguemos (ej. morado para vídeo, verde para documentos)."
    8. Corrección de Contraste (Modo Claro):"Arregla la visibilidad del modo claro para que los textos tengan alto contraste (negro/oscuro) y se lean bien sobre el fondo blanco, ya que los colores neón originales no se veían."

Explicacion del JavaScript más importante:

Scripts

1. El arranque (Setup)

Lo primero que hago es esperar a que cargue toda la web (DOMContentLoaded) para no tener fallos.

  • Aquí capturo los elementos del HTML (botones, zonas de arrastrar, el visor, etc.).
  • También arranco una función llamada loadPythonEngine. Lo que hace es cargar una librería externa (Pyodide) en segundo plano. Así, si luego metemos un archivo Python, el navegador ya estará listo para entenderlo.

2. El "Clasificador" (handleFile)

Esta es la función más importante. Cuando sueltas un archivo o lo seleccionas:

  1. Miro qué es: Saco la extensión (.jpg, .txt, .zip) y el tipo de archivo.
  2. Decido qué hacer (If/Else gigante):
    • Si es imagen/video/audio: Creo una URL temporal (con URL.createObjectURL). Es como engañar al navegador para que crea que el archivo está en internet, pero está en la memoria RAM, y se lo paso a una etiqueta <img> o <video>.
    • Si es Texto o Código: Uso el FileReader. Básicamente le digo: "lee el archivo como texto plano". Cuando termina, lo pego en el div del visor. Si es Markdown, hago unos reemplazos rápidos para que se vean las negritas y títulos.
    • Si es un ZIP: Aquí uso una librería externa (JSZip). Leo el archivo binario y me devuelve una lista de lo que hay dentro. Luego, con un bucle, voy pintando una lista <ul> para que parezca un árbol de carpetas.

3. El Reproductor de Música (Visualizer)

Para que se vean las barritas moviéndose con la música, usé la Web Audio API.

  • Conecto el archivo de audio a un "analizador".
  • Ese analizador me da números en tiempo real (frecuencias: graves, agudos...).
  • Uso un bucle infinito (requestAnimationFrame) que borra y pinta el <canvas> (el recuadro de dibujo) 60 veces por segundo. Dependiendo de lo fuerte que suene la música, pinto las barras más altas o más bajas.

4. El botón de "Ejecutar" (La Consola)

Aquí quise simular que es un editor de código de verdad.

  • Para JavaScript: Uso eval(). Es una función peligrosa en webs reales, pero aquí me sirve para coger el texto que hay en pantalla y ejecutarlo tal cual.
  • Para Python: Le paso el texto a la librería Pyodide que cargué al principio.
  • Para Java (El truco): Como el navegador no entiende Java, hice una trampa. Uso expresiones regulares (búsqueda y reemplazo) para cambiar System.out.println por console.log y int por let. Convierto el código Java a JavaScript "a lo bruto" y lo ejecuto. Parece Java, pero por debajo es JS.
