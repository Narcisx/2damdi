Monitoreo de Servidores

1. Prompts

Prompt: "Diseñar una arquitectura escalable para la orquestación de máquinas virtuales locales y ejecución remota de comandos mediante SSH secure channel."

Empecé el proyecto definiendo la arquitectura. Decidí que usaría Python (Flask) para el backend y que gestionaría las máquinas virtuales de forma local, asegurándome de que el sistema fuera escalable desde el principio.

2. Creando el control remoto

Prompt: "Desarrollar una API RESTful en Python que exponga endpoints para la gestión del ciclo de vida de VMs (start/stop/status) y monitorización de recursos (CPU/RAM) vía SSH."

Desarrollé la API para conectar el código con las máquinas. Logré crear los endpoints necesarios para encender y apagar los servidores, y usé SSH para monitorear el consumo de CPU y RAM en tiempo real.

3. Automatización del trabajo sucio

Prompt: "Scripting en Python para el despliegue desatendido (Unattended Install) de múltiples instancias de Ubuntu Server sobre VirtualBox."

Para ahorrar tiempo, creé un script en Python que automatiza la instalación de Ubuntu. Conseguí que el despliegue fuera totalmente desatendido, inyectando las credenciales automáticamente sin tener que tocar nada.

4. Visualización y emergencias

Prompt: "Implementar streaming de capturas de pantalla (Framebuffer dump) para visualización remota del estado de la consola y endpoint para Hard Reset."

Implementé un sistema de streaming de capturas (framebuffer) para poder ver la pantalla de las consolas remotamente. También añadí un botón de "Hard Reset" para reiniciar las máquinas a la fuerza si se quedaban congeladas.

5. Facilidad de acceso

Prompt: "Habilitar Cross-Origin Resource Sharing (CORS) y ejecución agnóstica del entorno para permitir el uso del dashboard como archivo local."

Configuré el CORS en el servidor. Esto me permitió abrir el dashboard directamente desde un archivo HTML local sin que el navegador bloqueara las peticiones al backend, ya que al principio lo hacía desde el localhost.

6. El cambio clave: Del SSH al teclado real

Prompt: "Quiero que escriba directamente como un teclado físico, no por consola SSH, para poder hacer login real."

Vi que por ssh no podia controlar la maquina virtual asi que empecé haciendo una inyección del teclado.

7. Mejorando la interfaz

Prompt: "Cuando pongo pantalla completa quiero tener a la derecha para mandar comandos."

Rediseñé la vista de pantalla completa. Dividí el espacio para mantener el panel de control accesible a la derecha mientras visualizo el servidor, mejorando mucho la usabilidad.

8. Optimización de fluidez

Prompt: "La pantalla va lenta, quiero que se vea casi en vivo."

Optimicé el algoritmo de captura y refresco. Reduje los tiempos de espera para conseguir una sensación de "tiempo real" en la transmisión de video de las máquinas.

9. Solución al problema crítico (Estabilidad)

Prompt: "Deja de funcionar al rato, se mezclan las pantallas y el segundo servidor no responde."

El servidor se volvía loco y colapsaba con tantas peticiones simultáneas. Lo arreglé implementando Semáforos (Thread Locking), obligando al sistema a ordenar las tareas en fila india para que no se atropellaran entre sí.

10. Integración de SSH y Consola Persistente

Prompt: "Quiero un botón SSH y que la salida no se borre."

Añadí un acceso directo a SSH y modifiqué la consola para que guarde el historial de comandos en lugar de borrarlos, permitiéndome revisar todo lo que he ejecutado.

11. Corrección de Bugs de Teclado

Prompt: "El guion sale como comilla simple."

Solucioné un fallo donde las teclas especiales salían cambiadas. Programé una traducción manual de códigos para que VirtualBox entienda exactamente lo que escribo desde mi teclado.

2. Funciones importantes

1. openFullscreen: Gestiono el video en pantalla completa usando un bucle inteligente que espera a que llegue una imagen antes de pedir la siguiente, evitando que se sature la red.

2. updateScreenshot: Actualizo las imágenes pequeñas del panel principal cada 5 segundos para ahorrar recursos cuando no estoy mirando una máquina específica.

3. sendCommand: Envío el texto que escribo al servidor simulando un teclado físico y fuerzo una recarga rápida de la imagen para ver el resultado casi al instante.

4. handleEnter: Detecto cuando pulso la tecla "Enter" para enviar el comando automáticamente, haciendo que se sienta como una terminal real.

5. updateServers: Sincronizo los colores y bloqueo los botones (Start/Stop) de la interfaz según si la máquina está realmente encendida o apagada.

6. fetchStats: Pido los datos de CPU y RAM por un canal separado del video; así, si el servidor tarda en calcularlos, la imagen no se congela.

7. sendCommand (Teclado): "Ciego". Envía teclas y espera ver el resultado en la imagen.

8. sendSSHCommand (Shell): "Síncrono". Envía el comando, espera a que el servidor responda con el texto (STDOUT/STDERR) y lo muestra en un <div> dedicado.

3. Tecnologías Utilizadas

  • Python: Lenguaje principal de orquestación.
    • Flask: Microframework web.
    • Paramiko: Cliente SSHv2.
    • Subprocess: Interfaz de comandos del sistema.
    • Threading: Gestión de concurrencia.
  • VirtualBox (VBoxManage): Hipervisor de virtualización.
  • Frontend: HTML5, CSS3 (Variables, Grid, Flexbox), JavaScript (ES6+, Fetch API).
